# Planning Skill

Comprehensive planning from requirements to implementation.

## Overview

Planning bridges the gap between initial requirements and code execution. This skill covers:
1. **Requirements Analysis** - Extracting and defining what to build
2. **Specification** - Documenting requirements as PRDs
3. **Implementation Planning** - Breaking down into actionable tasks

## When to Use

- User provides a feature request or task
- Requirements are vague or incomplete
- Need to break down complex work into tasks
- Creating PRD or technical specification

## Planning Workflow

1. **Analyze**: Understand the request and context
2. **Clarify**: Ask questions if information is missing
3. **Specify**: Create PRD or specification document
4. **Break Down**: Divide into bite-sized tasks
5. **Plan**: Create implementation plan with execution order

## Output

Plans should be saved to `docs/plans/YYYY-MM-DD-<feature-name>.md`
